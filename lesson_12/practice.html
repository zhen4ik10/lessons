CSS позиціонування

На головну

Різне CSS позиціонування дуже часто використовується при верстці сучасних макетів і відкриває для нас нові цікаві можливості розташування елементів у потоці. Але треба бути обережним та вміти поводитися з цими можливостями грамотно

Модуль позиціонування
Основна властивість - position, а також властивості розташування top, left, right, bottom, а також z-index

Значення властивості position:

  • static - (за замовченням) статичне позиціонування
  • relative - відносне позиціонування
  • absolute - абсолютне позиціонування
  • fixed - фіксоване позиціонування
  • sticky - липке позиціонування (гібридне поєднання значень static та fixed)

(!) Властивості top, left, right, bottom а також z-index працюють тільки зі значеннями position відмінним від static

z-index - управляє положенням об'єкту по осі Z (значення auto за замовченням) ближчим буде об'єкт який нижче по коду число від 0 до 9999 - порядковий номер на осі Z. Не впливає на позиціонування батьківського об'єкту у якого position відмінний від static

При позиціонуванні елементу вказавши значення для top, bottom, перестає працювати. Так само це працює з left та right. При цьому, якщо таки ж вказати обидві властивості то пріоритетними будуть: по Y top, а по X left. Гарною практикою є вказання значень лише для однієї властивості на вісь.

relative - відносне позиціонування Об'єкт позиціонується відносно свого поточного місця положення. (відносно свого положення в коді)

Для позиціонування об'єкту (з position: relative;) по вертикалі використовуючи відсотки неможливо без чіткої вказаної висоти для батьківського елементу Наприклад: top: 10% не спрацює без вказання висоти для батьківського елементу

absolute - абсолютне (безумовне) позиціювання.

  • 1 Разом з position: absolute; обов'язково використовуйте властивості для позиціонування (left або right, top або bottom)
  • 2 Об'єкт позиціонується відносно першого батьківського елементу, position якого відмінний від static.
  • 3 Об'єкт втрачає свою блочну сутність. Також втрачає своє положення в коді, та простір який він займає
  • 4 Не може вказувати для БЕМ-блоків
  • Для позиціювання об'єкту (position: absolute;) по вертикалі, використовуючи відсотки, відсоток буде братись від висоти батьківського елементу

fixed - фіксоване позиціонування.

  • 1 Разом з position: absolute; обов'язково використовуйте властивості для позиціонування (left або right, top або bottom)
  • 2 Об'єкт втрачає свою блочну сутність. Також втрачає своє положення в коді, та простір який він займає
  • 3 Завжди позиціонується відносно вікна браузера (в'юпорт).
  • 4 Не приймає участі у скролі
  • 5 Не може вказувати для БЕМ-блоків

sticky - це гібридне поєднання значень static та fixed Спочатку елемент веде себе як static але після того як скролл доходить до його кордону він стає fixed та прилипає до зазначеної позиції top або bottom

! ! ! Не працює, якщо у батьківськог об'єктах присутній overflow: hidden;
